**Date:** 2026-05-16
**Author:** Claude (Opus 4.7) with Maikel (MG)
**Status:** Approved by MG on 2026-05-16
**Replaces:** Implicit "v1 = everything" scope in `CONVENTIONS.md` v1.0
**Next:** Implementation plan via `writing-plans` skill

---

## 1. Purpose

Ship the first usable, visible version of Donum Dei: a small, polished plant-browser with filter, built on the chosen stack (Astro + React + TS + Tailwind + shadcn/ui), with manually-curated content for 7 medicinal plants (Letter A).

The goal is **a working, deployable static site in 3-5 sessions** that:

- proves the stack works end-to-end on Maikel's Mac,
- gives Maikel a concrete frontend codebase to learn from,
- can grow into v0.2+ (scrapers, map, calendar, quiz) without rewrite.

This is explicitly **not** the full vision from `CONVENTIONS.md` v1.0 — it is the smallest slice that ships.

---

## 2. Scope phasing

| Phase | Adds | Defers |
|-------|------|--------|
| **v0.1 (this spec)** | Card-grid landing, plant-detail pages, multi-select filter, DE/EN toggle, 7 hand-curated plants (Letter A), source attribution, educational disclaimer | Everything below |
| v0.2 | Scraper pipeline (Wikipedia, Wikidata, Wikimedia Commons), expansion to 20-50 plants | Map, calendar, quiz |
| v0.3 | Leaflet map + GBIF distribution data | Calendar, quiz, PWA |
| v0.4 | Seasonal/phenology calendar | Quiz, PubMed |
| v0.5+ | Quiz mode, PWA / offline, PubMed evidence layer, additional sources | — |

Each later phase gets its own brainstorm → spec → plan → build cycle. **This spec does not commit to v0.2+ details.**

---

## 3. Out of scope (v0.1)

Explicit non-goals, listed so the implementation plan does not drift:

- No scrapers of any kind (Wikipedia, GBIF, PubMed, Commons — all manual for v0.1).
- No map / Leaflet / GBIF integration.
- No seasonal calendar / phenology visualisation.
- No quiz / flashcards / scoring.
- No PWA / service-worker / offline support.
- No user accounts, login, notes, or favourites.
- No PubMed citation lookups.
- No folk-knowledge content (only public-domain classical quotes + Wikipedia-derived paraphrases).
- No SQLite, no backend, no API routes — pure static.
- No production deployment (local dev only; `npm run build` produces a working `dist/`, but no hosting decision in v0.1).

## 4. Functional design

### 4.1 Pages

| Route | Purpose | Components used |
|-------|---------|-----------------|
| `/` (redirects to `/de/` or `/en/` based on browser preference, fallback `/de/`) | Default entry — redirects to user's preferred language | Astro middleware |
| `/de/` , `/en/` | Landing page: filter bar + card grid | `<FilterBar>`, `<PlantCard>` (× 7) |
| `/de/plant/<slug>` , `/en/plant/<slug>` | Plant detail page | `<PlantDetail>`, `<SourceList>` |
| 404 | Friendly not-found | static |

`<slug>` is the Latin botanical name, lowercase, hyphenated (e.g. `equisetum-arvense`). Same slug works for both languages — language is chosen by URL prefix, slug stays stable.

### 4.2 Landing page (`/de/` and `/en/`)

- **Header**: site title `Donum Dei`, subtitle (DE: "Interaktive Heilpflanzen-Datenbank" / EN: "Interactive medicinal plants database"), language toggle (DE/EN) top-right.
- **Filter bar** (sticky, just below header):
  - Two filter groups:
    - **Verwendung / Use** (multi-select): Tee/Tea, Tinktur/Tincture, Salbe/Salve, Bad/Bath, Innerlich/Internal, Äußerlich/External.
    - **Saison / Season** (multi-select): Frühling/Spring, Sommer/Summer, Herbst/Autumn, Winter/Winter.
  - Live-filter: cards update on every change, no apply button.
  - Counter: "X von 7 Pflanzen" / "X of 7 plants".
  - "Filter zurücksetzen" / "Reset filters" button if any filter active.
- **Card grid**: responsive — 1 column on phone, 2 on tablet (≥ 768 px), 3 on desktop (≥ 1024 px).
- **Footer**: educational disclaimer, both languages always visible (or in active language only — decide during implementation; tested both).

### 4.3 Plant card (`<PlantCard>`)

- Image (thumbnail, ~400×300 px, optimised via Astro `<Image>`).
- Name (active language) + Latin name in italic parentheses below.
- 1-line teaser (active language, ~80 chars).
- Use-tags (compact icons or pills).
- Link to detail page.

### 4.4 Plant detail page (`<PlantDetail>`)

Sections, in order:

1. **Hero**: large image (~800×600 px optimised), full names (DE + EN + Latin always all three visible).
2. **Description** (active language; toggle shows other language if available, with "DE only" / "EN only" badge if not).
3. **Verwendung / Uses**: structured table or list — preparation form (Tee/Tinktur/...), what it is used for, internal/external.
4. **Saison / Season**: which months the plant is harvested or active (small inline calendar strip or just text).
5. **Klassische Zitate / Classical quotes** (if any): public-domain quotes from Hildegard, Künzle, Madaus, etc., with author and year. Original German preserved, English translation next to it where available.
6. **Sicherheit / Safety**: dosage warnings, contraindications, "äußerlich nur" / "external only" for plants like Arnika.
7. **Quellen / Sources** (`<SourceList>`): every fact source listed with link.
8. **Disclaimer footer**: educational use only, not medical advice.
9. **Back link** to landing page.

### 4.5 Language toggle (`<LanguageToggle>`)

- Two buttons "DE" and "EN" in header, current language highlighted.
- Click swaps URL prefix (`/de/...` ↔ `/en/...`) without losing position.
- Persists choice in `localStorage` under `donum_dei_lang`.
- On first visit: detect `navigator.language`, fallback DE.

### 4.6 Disclaimer

Both languages:

> DE: *Diese App dient ausschließlich der Bildung. Sie ersetzt keine medizinische Beratung. Bei Beschwerden Arzt oder Apotheker konsultieren.*
> EN: *This application is for educational purposes only. It does not replace medical advice. Consult a physician or pharmacist for health concerns.*

Shown in active language at footer of every page.

## 5. Data model

### 5.1 JSON schema (`docs/data_schema.md` — to be written before any plant JSON)

One file per plant: `src/data/plants/<slug>.json`. Indicative shape:

```json

  "names": {
    "de": "Acker-Schachtelhalm",
    "en": "Field Horsetail",
    "latin": "Equisetum arvense"
  },
  "family": { "de": "Schachtelhalmgewächse", "en": "Horsetail family", "latin": "Equisetaceae" },
  "description": {
    "de": "...",
    "en": "..."
  },
  "uses": [
    {
      "form": "tea",
      "target": ["urinary", "connective_tissue"],
      "internal_external": "internal",
      "description": { "de": "...", "en": "..." },
      "source_ids": ["src_1", "src_3"]
    }
  ],
  "season": {
    "active_months": [4, 5, 6, 7, 8],
    "harvest_part": { "de": "junge Triebe", "en": "young shoots" }
  },
  "safety": {
    "warnings": { "de": "...", "en": "..." },
    "external_only": false
  },
  "classical_quotes": [
    {
      "author": "Künzle",
      "year": 1911,
      "license": "PD",
      "text_de": "...",
      "text_en": "..."
    }
  ],
  "sources": [
    {
      "id": "src_1",
      "type": "wikipedia",
      "url": "https://de.wikipedia.org/wiki/Acker-Schachtelhalm",
      "accessed": "2026-05-16"
    }
  ],
  "image": {
    "filename": "equisetum-arvense.jpg",

    "author": "Photographer name",
    "source_url": "https://commons.wikimedia.org/wiki/File:..."
  }
}
```

The exact schema is finalised in `docs/data_schema.md` during implementation; the structure above is the design target.

### 5.2 Image storage

- Files in `public/images/plants/<slug>.jpg` (Commons download).
- Sidecar metadata in JSON (`image` block above) — license + author + source URL travels with the image.
- Astro `<Image>` component generates responsive sizes at build time (`400w`, `800w`, `1200w`).

### 5.3 The 7 v0.1 plants (Letter A)

Hand-curated, all classical-medicinal, all with abundant CC photography on Wikimedia Commons and good Wikipedia coverage in DE+EN:

| # | DE | Latin | Primary use | Notes |
|---|----|-------|-------------|-------|
| 1 | Acker-Schachtelhalm | *Equisetum arvense* | Tee (Niere, Bindegewebe) | Common, distinctive, well-studied |
| 2 | Schafgarbe | *Achillea millefolium* | Tee, Tinktur (Verdauung, Wunden) | Classic, abundant |
| 3 | Alant | *Inula helenium* | Tee, Tinktur (Husten) | Hildegard mentions |
| 4 | Andorn | *Marrubium vulgare* | Tee (Atemwege, Verdauung) | Pflanze des Jahres 2018 (Germany) |
| 5 | Anis | *Pimpinella anisum* | Tee, Gewürz (Verdauung) | Familiar to everyone |
| 6 | Arnika | *Arnica montana* | Tinktur, Salbe (Prellungen) | **External only** — important safety note |
| 7 | Augentrost | *Euphrasia officinalis* | Tee, Augenbad (Augen) | Distinctive name + use |

If during writing it turns out one of these is genuinely hard to source-attribute or has license-fragile imagery, the implementation plan may substitute (e.g. Ackerstiefmütterchen *Viola arvensis*).

## 6. Technical design

### 6.1 Stack (unchanged from `CONVENTIONS.md` v1.0)

| Layer | Tech | Why |
|-------|------|-----|
| Framework | Astro 4.x | Static-first, deploys anywhere |
| Islands | React 18 | Filter is interactive; rest is static |
| Language | TypeScript 5.x strict | Type-safe plant data |
| Styling | Tailwind 3.x | Fast, consistent, mobile-first |
| Components | shadcn/ui | Pretty defaults, customisable |
| i18n | `@astrojs/i18n` | URL-prefix routing, built into Astro |
| Data | JSON files in `src/data/plants/` | Version-controllable, no DB needed |

### 6.2 Component map

| Component | Type | Purpose |
|-----------|------|---------|
| `<PlantCard>` | Astro | Static card in grid |
| `<PlantDetail>` | Astro | Static detail page body |
| `<FilterBar>` | React island | Interactive filter |
| `<LanguageToggle>` | React island | Persistent language swap |
| `<SourceList>` | Astro | Static source listing |
| `<Disclaimer>` | Astro | Footer disclaimer |
| `<Image>` (Astro built-in) | Astro | Responsive image |

Only two React islands. Everything else is server-rendered at build time.

### 6.4 i18n strategy

- Astro built-in i18n (`@astrojs/i18n`), URL prefix routing (`/de/...`, `/en/...`).
- Default locale: `de`. Fallback locale: `de`.
- UI strings live in `src/lib/i18n.ts` (typed dictionary, both languages).
- Plant data carries DE + EN content in every text field. Missing language → badge + fallback to other language (no broken pages).

### 6.5 Filter logic

- Filter state lives in `<FilterBar>` React island, passes filter criteria up via a small `nanostores` or React-context store (decide during implementation, simplest wins).
- Filtering is client-side over the 7 plants (trivial set size).
- Multi-select within a group = OR; across groups = AND. Example: `[Tee, Tinktur] × [Sommer]` = plants used as tea OR tincture AND active in summer.

### 6.6 Build / dev

- `npm run dev` → `localhost:4321` with hot reload.
- `npm run build` → `dist/` static output. Verified to load locally via `npm run preview`.
- No deployment in v0.1 (no hosting decision needed).

## 7. Setup tasks (the implementation plan will sequence these)

Listed for reference; the `writing-plans` skill will turn these into ordered, checkable tasks.

1. **Node version check** — confirm Node ≥ 20 LTS on Mac; if missing, install via `brew install node@20`.
2. **`git init`** local repo (no remote), add `.gitignore` for `node_modules/`, `dist/`, `.astro/`, `.DS_Store`.
3. **Astro scaffold** — `npm create astro@latest -- --template minimal --typescript strict`.
4. **Astro integrations** — add React, Tailwind, MDX (optional), sitemap, i18n via `npx astro add`.
5. **shadcn/ui init** — `npx shadcn@latest init` with sensible defaults.
6. **Write `docs/data_schema.md`** — finalise the plant JSON schema.
7. **Write 7 plant JSONs** — Claude drafts from PD knowledge + Wikipedia, MG reviews each.
8. **Download 7 plant images** from Wikimedia Commons (CC-licensed), save with sidecar metadata.
9. **Implement components** in this order: `Disclaimer` → `LanguageToggle` → `PlantCard` → `FilterBar` → `PlantDetail` → `SourceList`.
10. **Implement pages**: landing (DE + EN), detail (`[slug].astro`, DE + EN), index redirect.
11. **Add `loadPlants.ts`** loader + schema validation.
12. **Style polish** — pick a calm, herbal-ish colour scheme (green/cream), responsive test on phone.
13. **Build test** — `npm run build` + `npm run preview` works.
14. **First git commit** with everything above.
15. **Update `SESSION_STATE.md`, `TODO_Donum_Dei.md`, `CONVENTIONS.md` (→ v1.1 with scope cut)**.

## 8. Success criteria

v0.1 is **done** when:

- [ ] `npm run dev` shows landing page with 7 plant cards in card grid.
- [ ] Filter bar narrows the visible plants live, multi-select works, counter updates.
- [ ] DE/EN toggle swaps URL prefix, content language, and persists in localStorage.
- [ ] Each plant has a detail page reachable via card click, with image, description, uses, classical quote, sources, disclaimer.
- [ ] Every fact carries a source attribution.
- [ ] Educational disclaimer is visible on every page in active language.
- [ ] `npm run build` produces a working `dist/` that `npm run preview` serves correctly.
- [ ] Site is responsive on phone (Maikel tests on his actual phone).
- [ ] `git log` shows clean commit history of the build process.

## 9. Risks and mitigations

| Risk | Mitigation |
|------|-----------|
| Maikel does not have Node ≥ 20 → broken setup | First plan step: explicit `node --version` check + brew install if missing |
| shadcn/ui defaults look too generic / not "Donum Dei" enough | Reserve a polish step in plan (Step 12) — colour palette, typography (e.g. serif headings for herbal feel) |
| Plant content writing turns into a research swamp | Cap each plant at ~30 min of writing; substitute with a different A-plant if a particular one is too thin |
| Image attribution gets sloppy | Sidecar `.meta.json` enforced from the first image; loader validates presence |
| Astro/React island wiring is unfamiliar territory for Maikel | Implementation plan uses small steps + verification between steps so Maikel can read the diff each time |
