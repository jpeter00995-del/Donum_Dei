**Date:** 2026-05-17
**Author:** Claude (Opus 4.7, 1M context) with Maikel (MG)
**Status:** Awaiting MG review

---

## 1. Purpose

Add an "Indoor Growing" area to Donum Dei: a dedicated page where users can find plants suitable for growing inside the home, plus care instructions for each. Extends the existing project from "medicinal-plant reference" toward "practical companion for the home gardener" — without breaking the scientific tone or the educational disclaimer.

Two distinct content groups live in this area:

1. **Heilen + Küche indoor:** Medicinal/culinary plants that grow well on a windowsill (Aloe vera, Stevia, Tulsi, indoor herbs, etc.). Some are new to the DB, some are retrofits of existing plants.
2. **Nützliche Indoor-Pflanzen:** Not medicinal, but with a concrete practical benefit at home — air-purifying (NASA Clean-Air classics), pest-repelling (mosquito/insect repellents), humidifying (against heating dryness), and night-oxygen producers (CAM photosynthesis, good for the bedroom). Each plant is tagged with its specific benefit via `purpose` enum.

Interactivity: a 3-question mini-quiz ("Welche Pflanze passt zu dir?") plus standard filters (room, light, pet-safe, purpose).

## 3. Out of scope (v0.8)

Explicit non-goals to prevent drift:

- No backend, no user accounts, no persistent favourites server-side.
- No AR / camera-based light measurement (mobile-only, needs native APIs).
- No push notifications or watering reminders (requires service-worker push subscriptions + backend).
- No automated import of pet-toxicity data from ASPCA (manual curation for v0.8 — 30 plants max).
- No "what's wrong with my plant" diagnostic flow.
- No pest/disease database.
- No purchase links or affiliate integration.
- Cannabis / illegal-in-some-jurisdictions plants stay out (user request).

## 4. Data schema

### 4.1 New optional field on every plant JSON

```json
"indoor_growing": {
  "suitable": true,
  "purpose": ["medicinal", "edible"],
  "rooms": ["kitchen", "living_room"],
  "light": "bright_indirect",
  "water_frequency": "weekly",
  "soil": {
    "de": "Standard-Kräutererde, durchlässig",
    "en": "Standard herb compost, well-draining"
  },
  "pot_size_cm": 15,
  "difficulty": 1,
  "pet_safe": true,
  "tips": {
    "de": [
      "Topf mit Drainage-Löchern wählen — Staunässe ist der häufigste Killer.",
      "Im Winter heller stellen oder Pflanzenlampe — sonst werden Triebe lang und blass.",
      "Alle 4-6 Wochen Flüssigdünger im Sommer."
    ],
    "en": [
      "Choose a pot with drainage holes — waterlogging is the most common killer.",
      "Move to a brighter spot in winter or use a grow light — otherwise shoots become long and pale.",
      "Liquid fertiliser every 4-6 weeks in summer."
    ]
  }
}
```

### 4.2 Controlled vocabularies

These are enums enforced by `validatePlant.ts`. Unknown values fail validation.

- `purpose`: array, one or more of `medicinal` | `edible` | `air_purifying` | `pest_repelling` | `humidifying` | `night_oxygen` | `ornamental`
- `rooms`: array, one or more of `kitchen` | `living_room` | `bathroom` | `bedroom` | `balcony`
- `light`: single string, one of `direct_sun` | `bright_indirect` | `partial_shade` | `low_light`
- `water_frequency`: single string, one of `daily` | `every_few_days` | `weekly` | `sparse`
- `difficulty`: integer 1 | 2 | 3 (1 = beginner, 3 = experienced)
- `pet_safe`: boolean (cats + dogs — single field, no per-species split for v0.8)

### 4.3 Free-text fields

- `soil`: short bilingual string. Free-form because soil mixes vary too much for a useful enum.
- `pot_size_cm`: integer, recommended starting pot size in centimetres.
- `tips`: bilingual arrays of 1-3 short bullet-point strings.

### 4.4 Required vs. optional

Entire `indoor_growing` block is optional at the plant level. Plants without it do not appear in the indoor area.

When `indoor_growing` IS present, these are required: `suitable`, `purpose` (≥1 entry), `rooms` (≥1 entry), `light`, `water_frequency`, `difficulty`, `pet_safe`. Optional: `soil`, `pot_size_cm`, `tips`.

This lets us add a quick "yes this plant is indoor-suitable" marker without forcing full content immediately.

## 6. UI design

### 6.1 New page: `/de/zuhause-anbauen` and `/en/grow-at-home`

Layout top-to-bottom:

1. **Hero block** — title + 1-2 sentence intro ("Kein grüner Daumen nötig…").
2. **Quiz-Block** *(Phase 2)* — collapsed by default, "🌱 Welche passt zu dir? (3 Fragen)" CTA.
3. **Filter-Bar** *(Phase 2)*:
   - Room chips: Küche / Wohnzimmer / Bad / Schlafzimmer / Balkon
   - Light slider (3 steps: dunkel / mittel / sonnig)
   - Pet-safe toggle
   - Purpose chips: Heilen / Küche / Luftreiniger / Mückenschutz / Befeuchter / Nacht-O₂ (multi-select; wraps on mobile)
4. **Cards-Grid** — all plants with `indoor_growing.suitable = true`, default sort by `difficulty` ascending (easy first). In Phase 1 there is no filter; the grid simply shows everything.

Card content:
- Plant photo (existing `image.filename`)
- Name (DE/EN per current language)
- Difficulty stars (★☆☆ / ★★☆ / ★★★)
- Quick icons: ☀️/🌤/🌥 light + 💧/💧💧/💧💧💧 water + 🐾 if pet-safe
- Purpose mini-badges: 🌿 (heilen) 🍃 (küche) 💨 (luftreiniger)

Clicking a card → existing plant detail page (`/de/plant/<slug>/`).

### 6.2 Quiz flow (Phase 2)

3 sequential questions with progress dots `●●○`:

| # | Question (DE) | Options |
|---|---------------|---------|
| 1 | "Wo soll deine Pflanze stehen?" | Küche / Wohnzimmer / Bad / Schlafzimmer / Balkon |
| 2 | "Wie viel Licht kommt da hin?" | Sonniges Fenster / Helles Zimmer / Eher dunkel |
| 3 | "Wie oft denkst du ans Gießen?" | Vergesslich / Normal / Gärtnerisch-pingelig |

**Match algorithm:** simple weighted score, all client-side, no LLM:

```
score = 0
if plant.rooms contains answer1:                    score += 3
if plant.light is in compatible_set(answer2):       score += 2
if plant.water_frequency is in compat_set(answer3): score += 2
# Easier plants get a small boost within equal-match groups:
score += (4 - plant.difficulty)
```

Mappings:

- `answer2` (light) → compatible plant.light values:
  - `sonnig` → `{direct_sun, bright_indirect}`
  - `hell` → `{bright_indirect, partial_shade}`
  - `dunkel` → `{partial_shade, low_light}`
- `answer3` (watering habit) → compatible plant.water_frequency values:
  - `vergesslich` → `{sparse, weekly}`
  - `normal` → `{weekly, every_few_days}`
  - `pingelig` → `{daily, every_few_days, weekly, sparse}` (any — pingelig user can handle anything)

Difficulty is implicit: easier plants (lower `difficulty`) always score higher within the same room/light/water match group, so beginners get easy plants surfaced first without needing a 4th question.

Show top-5 by score. If fewer than 5 plants have any score > 0, show "Wenig Treffer — vielleicht magst du diese probieren?" + top-3 absolute.

Result-screen: 5 cards + reset button. Cards link to detail pages.

## 7. Content sourcing strategy

### 7.1 Pflege-Karte fields (light, water, soil, pot_size, difficulty)

Sourced by Claude from a combination of:

- **Wikipedia DE/EN** Pflege/Anbau sections (most plants have these)
- Common Indoor-Gardening reference knowledge (Royal Horticultural Society, common houseplant care guides — concepts, not quotes)

Recorded as plant-level content; not separately attributed in `sources[]`. The page-level disclaimer covers them.

### 7.2 Pet-safe field

Sourced from **ASPCA Toxic and Non-Toxic Plants Database** (https://www.aspca.org/pet-care/animal-poison-control/toxic-and-non-toxic-plants).

Each plant with `pet_safe` set gets an extra entry appended to its existing `sources[]` array:

```json
{
  "id": "src_3",
  "type": "database",
  "title": "ASPCA Toxic/Non-Toxic Plants Database",
  "url": "https://www.aspca.org/pet-care/animal-poison-control/toxic-and-non-toxic-plants/<slug-or-search>",
  "accessed": "2026-05-17"
}
```

### 7.3 Freitext-Tipps

Claude-generated bullet points from general indoor-gardening knowledge. Disclaimer in detail-page section makes the type clear. Maikel can edit any tip in the plant JSON.

No `classical_quotes` are added for indoor tips — those remain reserved for Hildegard/Künzle/Madaus medicinal quotes.

## 10. Risks and mitigations

| Risk | Mitigation |
|------|------------|
| Wikipedia images missing for some new plants (esp. Aerva lanata, Houttuynia) | Existing scraper fallback to `de.wikipedia.org/wiki/<title>` URL when Commons metadata missing; we patched 3× in v0.7 the same way |
| ASPCA URLs hard to construct per-species | Use search URL form rather than deep link; verify manually for 30 plants |
| Pet-safe data ambiguous for some species (e.g. "mildly toxic") | Strict binary: any non-trivial toxicity → `pet_safe: false`. Detail-page warning text explains nuance |
| Quiz score gives 0 hits for unusual combinations (e.g. balcony + low_light + pingelig) | Fallback "top-3 absolute" path described in §6.2 |
| Donum-Dei "medicinal" identity diluted by non-medicinal plants | All non-medicinal plants are tagged with a concrete-benefit `purpose` (`air_purifying`, `pest_repelling`, `humidifying`, `night_oxygen`) — never with `ornamental` alone, so every plant in the DB serves a documented function. Detail page does NOT show the "Verwendung" (medicinal-uses) section for plants without `purpose: medicinal`; instead it shows "Praktischer Nutzen" with the relevant purpose tags expanded into 1-line descriptions |
