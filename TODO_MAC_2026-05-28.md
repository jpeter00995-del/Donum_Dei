## 🎯 1. HAUPTAUFGABE — Plant-Bild-Optimierung (60-90 min)

**Warum**: Lighthouse Performance 31/100. `dist/` ist 11.7 MB primär wegen `src/public/images/plants/`. `thuja-occidentalis.jpg` allein ist **4 MB**. Top-10-Images konvertieren zu WebP + responsive Sizes bringt Performance vermutlich auf 60-70/100.

### 1.1 — Worst Offenders identifizieren
```sh
du -h public/images/plants/*.jpg | sort -hr | head -15
```
Erwartet: thuja-occidentalis, equisetum-arvense, pimpinella-anisum, ein paar 800KB-1MB-Files.

### 1.2 — Option A: Bulk-Convert mit `cwebp` (CLI, schnellste Variante)
```sh
brew install webp                    # wenn nicht installiert
cd public/images/plants

# Backup zuerst (P10!)
mkdir _backup_pre_webp_2026-05-28
cp *.jpg _backup_pre_webp_2026-05-28/

# Convert mit Quality 75 (Lighthouse-Empfehlung Standard)
for f in *.jpg; do
  cwebp -q 75 "$f" -o "${f%.jpg}.webp"
done

# Check Größen
du -sh *.webp | head -5
du -sh *.jpg | head -5    # zum Vergleich
```

Wenn WebP kleiner ist (sollte 60-80% Einsparung sein):
```sh
# Im Code: img-tags auf .webp umstellen
# Suchen wo plant.jpg referenziert wird:
grep -rn "\.jpg" src/components/ src/pages/ src/lib/ | head -5
```

Vermutlich in `PlantDetail.astro` und FilterBar.tsx wo das Plant-Bild gerendert wird. Path-Pattern `/images/plants/<slug>.jpg` → `/images/plants/<slug>.webp`.

### 1.3 — Option B: Astro-Image-Pipeline (sauberer, aber mehr Setup)
```sh
# astro:assets nutzt Sharp unter der Haube, generiert WebP + AVIF + responsive Sizes
# Doku: https://docs.astro.build/en/guides/images/
```
Schritte:
1. Bilder von `public/images/plants/` → `src/assets/plants/`
2. In Components: `import { Image } from 'astro:assets'`
3. `<Image src={...} alt={...} widths={[320, 640, 960]} formats={['webp']} />`
4. Astro generiert beim Build automatisch.

→ **Empfehlung**: Erst Option A (Quick-Win, 30 min), dann später Astro-Image als Refactor.

### 1.4 — Verifikation
```sh
npm run build               # 481 Seiten clean?
npm test                    # 285/285 grün?
du -sh dist/                # sollte deutlich kleiner sein (zielen auf < 30 MB)
```
